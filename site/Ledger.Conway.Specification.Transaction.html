    Transactions¶

    A transaction consists of a transaction body, a collection of witnesses and some optional auxiliary data.

    This section collects the era-dependent types and functions that are used to define transactions in the Conway era.

    {-# OPTIONS --safe #-}
open import Ledger.Core.Specification.Abstract
open import Ledger.Core.Specification.Transaction

module Ledger.Conway.Specification.Transaction
  (txs : _) (open TransactionStructure txs)
  (abs : AbstractFunctions txs)
  where

open import Ledger.Prelude
open import Ledger.Conway.Specification.Certs govStructure

    Transaction Types¶

    We now describe the key components of the transaction body in the Conway era.

    • txIns denotes a set of transaction inputs; each input consists of a transaction id and an index that points to a unique output from a previous transaction.

    • txOuts denotes an indexed collection of transaction outputs; each output consists of an address and a coin value.

    • txFee denotes a transaction fee to be added to the fee pot.

    • txId denotes the hash of the serialized form of the transactionthe form in which the transaction is included in a block.

    In addition to these, the Conway era introduces some new fields to the transaction body.

    • txGovVotes denotes the list of votes for governance actions.

    • txGovProposals denotes the list of governance proposals.

    • txDonation denotes the amount of Coin to donate to treasury, e.g., to return funds to the treasury after a governance action.

    • currentTreasury denotes the current value of the treasury; this serves as a precondition to executing Plutus scripts that access the value of the treasury.

    record TxBody : Type where
  field
    txIns                :  TxIn
    refInputs            :  TxIn
    collateralInputs     :  TxIn
    txOuts               : Ix  TxOut
    txId                 : TxId

    txFee                : Fees
    txWithdrawals        : Withdrawals
    txVldt               : Maybe Slot  Maybe Slot
    txADhash             : Maybe ADHash
    txDonation           : Donations
    txGovVotes           : List GovVote
    txGovProposals       : List GovProposal
    txNetworkId          : Maybe Network
    currentTreasury      : Maybe Treasury
    mint                 : Value
    reqSignerHashes      :  KeyHash
    scriptIntegrityHash  : Maybe ScriptHash

  field TxBodyOf : A  TxBody

record HasDCerts {a} (A : Type a) : Type a where
  field DCertsOf : A  List DCert
open HasDCerts ... public

record HasGovProposals {a} (A : Type a) : Type a where
  field GovProposalsOf  : A  List GovProposal
open HasGovProposals ... public

record HasTxId {a} (A : Type a) : Type a where
  field TxIdOf    : A  TxId
open HasTxId ... public

instance
  HasDonations-TxBody : HasDonations TxBody
  HasDonations-TxBody .DonationsOf = TxBody.txDonation

  HasGovProposals-Tx .GovProposalsOf = TxBody.txGovProposals  TxBodyOf

  HasWithdrawals-Tx : HasWithdrawals Tx
  HasWithdrawals-Tx .WithdrawalsOf = WithdrawalsOf  TxBodyOf

  HasTxId-Tx .TxIdOf = TxBody.txId  TxBodyOf

  HasDonations-Tx : HasDonations Tx
  HasDonations-Tx .DonationsOf = DonationsOf  TxBodyOf

    Transaction Functions¶

    refScripts : Tx  UTxO   Script
refScripts tx utxo =
  mapPartial (proj  proj  proj) (range (utxo  (txIns  refInputs)))
  where open Tx; open TxBody (tx .body)

txscripts : Tx  UTxO   Script
txscripts tx utxo = scripts (tx .wits)  refScripts tx utxo
  where open Tx; open TxWitnesses

lookupScriptHash : ScriptHash  Tx  UTxO  Maybe Script
lookupScriptHash sh tx utxo = lookup? m sh
  where m = setToMap (map < hash , id > (txscripts tx utxo))
